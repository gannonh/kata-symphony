# Symphony Quality Score

Last reviewed: 2026-03-05

## Purpose

Define a lightweight scorecard to track engineering quality as implementation progresses.

## Dimensions

Each dimension is scored 0-5.

1. Spec conformance
2. Test coverage of required behaviors
3. Reliability and recovery behavior
4. Security and safety posture
5. Operability and observability
6. Documentation legibility for agents and humans

## Current Baseline

- Spec conformance: 1 (spec exists; implementation not started)
- Test coverage: 0
- Reliability: 0
- Security: 0
- Operability: 0
- Documentation legibility: 2 (harness baseline scaffold in place)

## Exit Criteria For MVP

- No dimension below 3
- Spec conformance and test coverage both at least 4
